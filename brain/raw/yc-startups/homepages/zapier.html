Connect to OpenClaw

Paste the following prompt into a new OpenClaw conversation and follow the instructions:

Set up Zapier MCP for OpenClaw

Setup steps:
1. Install mcporter if you don't have it: npx -y mcporter
2. Add the Zapier MCP server: mcporter config add zapier --url "https://mcp.zapier.com/api/v1/connect" --client-name "openclaw"
3. Verify the connection: mcporter list zapier --schema

Once you've confirmed the connection works and tools are listed, call the "Zapier MCP onboarding" skill. This will walk me through adding apps and configuring my tools step by step. Follow the skill's instructions and dialogue with me until onboarding is complete.

AI governance and security that scales with you

Control what runs, empower teams to build, and stay on top of everything in real-time.

Control

Define exactly what runs

Most platforms give you a binary choice: allow an app or block it. Zapier goes deeper — endpoint-level restrictions, corporate-only authentication, and the option to route AI through your own infrastructure. The more precisely you control, the more confidently you can say "yes" to builders.
Features
Define exactly which actions are allowed in each app so teams can move fast without compromising sensitive systems.
Run AI through your own infrastructure, including Amazon Bedrock, so you can adopt AI automation while keeping your existing security, compliance, and access controls in place.
Use company-owned app connections managed by IT to ensure consistent access and keep automations running as teams change.
Block personal accounts from connecting to business systems to prevent data from leaving your environment.
Allow or block apps across your workspace so teams only use tools that meet your security standards.
Delegation

Give teams their own space

Workspaces let you delegate authority without losing oversight. Each team gets their own environment with inherited guardrails. They build freely and you see everything. No more choosing between access and accountability.
Features
Create dedicated environments for each team with the right apps, connections, and controls, so they can build independently while staying within IT-defined guardrails and security standards.
Start from approved templates, so every build begins with the right structure, tools, and safeguards.
Control who can view, build, or manage workflows by assigning role-based permissions.
Automate user provisioning through your identity provider so access stays current as teams change.
Visibility

Gain full insight into every workflow

Every AI action is logged, every workflow is documented, and every output is checked, so you always know what's happening.
Features
Access audit data programmatically through an API so you can query, analyze, and report on automation activity at scale.
Add built-in safety checks that block sensitive data and detect risky inputs before AI outputs reach your systems.
Send real-time workflow data to tools like Datadog or Splunk for centralized monitoring.
Automatically generate documentation for every workflow, so teams can easily understand and audit processes.
