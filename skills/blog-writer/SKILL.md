# Blog Writer

Write blogs with a clear purpose, fitting structure, and strong stylistic model.

## Workflow

1. Identify what kind of blog the user is writing and why.
2. Read `../../references/principles.md`.
3. Read the relevant brain source:
   `../../brain/sources/paul-graham/README.md` for founder essays, or
   `../../brain/sources/lenny-newsletter/README.md` for product/growth/operator
   writing.
4. Read `references/instructions.md`.
5. Choose the expert-style model that best fits the job.
6. Draft or outline using that model without copying the person directly.
7. Edit for clarity, originality, and usefulness.

## Output

If the request is broad, first state:

- Blog type
- Purpose
- Audience
- Chosen style model
- Why that model fits

Then write the outline or draft.
